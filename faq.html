FAQ

Clear answers for the current iPad build, with no incense-clouded feature promises.

What kind of game is Shramana Project?

It is a side-view puzzle game inspired by classic crowd-routing puzzles. Monks spawn automatically, the player assigns limited abilities, and most levels are solved by sacrificing many monks so one monk reaches the gate.

What platforms are supported?

The project is iPad-only for iPadOS 17 and later. It is designed for landscape left and landscape right orientation.

How many levels are planned in this build?

The canonical campaign has 20 levels across four chapters, with separate chapter metadata and one JSON level file per level.

Is Mudra playable?

Mudra is scaffolded in code, JSON decoding, icons, and animation lookup, but it is not required by the shipped campaign content in this implementation pass.

Does the app need an account or network connection?

The current gameplay architecture is local. Save data is stored on device and the repo does not include account, networking, cloud sync, or telemetry systems.

Where are final visuals coming from?

Runtime assets use generated PixelLab art: monk sprites, ability icons, chapter backdrops, environment objects, and UI assets.
