Support

Help for playtesting, build verification, and future release support.

Contact

Use the project repository issue tracker or release contact address once the public store listing is created. Until then, support requests should include iPad model, iPadOS version, level number, and a short description of what happened.

Useful fixes

  • Restart the level if a tutorial prompt is dismissed before reading.
  • Use 1x speed for timing-heavy platforms, portals, and stop-chant puzzles.
  • Open the pause diagnostics sheet in development builds when reporting content or asset loading issues.

Release support promise

The site avoids claiming cloud sync, accounts, mod support, or network features because those systems are out of scope for the current project. Support copy will be updated when the App Store listing and final contact route exist.
